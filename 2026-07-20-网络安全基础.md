# 2026-07-20 网络安全基础学习

## 一、基础概念名词

### DNS（Domain Name System，域名系统）
将域名（如 `www.example.com`）解析为 IP 地址的分布式数据库系统，是互联网的基础服务之一。

### CDN（Content Delivery Network，内容分发网络）
通过在全球部署边缘节点服务器，将网站内容缓存到离用户最近的节点，加速用户访问速度，同时减轻源站压力。

### Web 网站的构成
一个典型的 Web 应用通常由以下层次组成：
- **操作系统层**：服务器运行的操作系统（Linux/Windows Server）
- **Web 服务器层**：Nginx / Apache / IIS 等，负责接收和处理 HTTP 请求
- **应用层**：后端代码（Java/Python/PHP/Go 等）和前端代码（HTML/CSS/JavaScript）
- **数据库层**：MySQL / PostgreSQL / MongoDB 等，存储业务数据

## 二、针对 Web 各层的攻击方式

| 目标层次 | 常见攻击方式 | 说明 |
|---------|-------------|------|
| 操作系统层 | 漏洞利用、权限提升、DDoS | 利用 OS 未修补的漏洞获取服务器控制权 |
| Web 服务器层 | 配置漏洞利用、目录遍历 | 利用服务器配置不当访问敏感文件 |
| 源码/应用层 | SQL 注入、XSS、CSRF、文件上传漏洞 | 针对代码逻辑缺陷进行攻击 |
| 数据库层 | SQL 注入、弱口令爆破 | 直接或间接攻击数据库 |

## 三、后门（Backdoor）

后门是指攻击者在成功入侵系统后，为了保持长期访问权限而留下的隐蔽通道。常见形式：
- **Webshell**：放置在 Web 服务器上的恶意脚本文件
- **定时任务/计划任务**：通过系统定时任务反弹 shell
- **隐蔽账号**：创建不易被发现的管理员账号
- **SSH 公钥植入**：将自己的公钥写入目标服务器 authorized_keys

## 四、HTTP/HTTPS 请求

### 请求的组成
一个 HTTP 请求主要包含以下部分：

1. **请求行**：请求方法（GET/POST/PUT/DELETE 等）+ URL + HTTP 版本
2. **请求头（Headers）**：
   - `Host`：目标主机
   - `User-Agent`：客户端标识
   - `Cookie`：会话信息
   - `Content-Type`：请求体格式
   - `Authorization`：认证信息
3. **空行**：分隔 Headers 和 Body
4. **请求体（Body）**：POST/PUT 请求携带的数据

### HTTPS 与 HTTP 的区别
- HTTP 明文传输，HTTPS 通过 TLS/SSL 加密
- HTTPS 需要 CA 签发证书

### 查看请求的方法
- 浏览器开发者工具（F12）→ Network 面板
- 命令行工具：cURL
